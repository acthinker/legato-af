Run Processin App Sandbox

Tool used to execute a process in a running application's sandbox.

The executed process will retain the standard streams of the terminal that calls this tool. The 5 terminals environment variable will also be passed to the executed process. 6

The executed process will not be monitored by the Legato Supervisor. However, the process will 8 run as the same user as the application and thus will be killed when the application stops. 9

The executable and all required libs, devices, etc. must already be in the application's sandbox 11 before the process can be started.

Prints a generic message on stderr so that the user is aware there is a problem, logs the 26 internal error message and exit.

To run a process in a application's sandbox from the command line:

Usage: 

execInApp appName [OPTIONS] execPath [ARGS]

Options:

-f
Prints updated memory usage details for the process.

The usage is:

SYNOPSIS:

DESCRIPTION: Runs the specified executable in the specified application's sandbox.

The appName is the name of the running application that the process should start in. The appName cannot start with a '-'.

The execPath is the path in the sandbox to the executable file that will be executed. The execPath cannot start with a '-'.

The excutable and all required libraries, resources, etc. must exist in the application's sandbox.

The executed process will inherit the environment variables and file descriptors of the terminal.

OPTIONS: –procName=NAME Starts the process with NAME as its name. If this option is not used the executable name is used as the process name.

–priority=PRIORITY Sets the priority of the process to PRIORITY. PRIORITY must be either 'idle', 'low', 'medium', 'high', 'rt1', 'rt2'...'rt32'.

–help Display this help and exit.

ARGS: This is a list of arguments that will be passed to the executed process.
